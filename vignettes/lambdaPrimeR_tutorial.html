Introduction

External dependencies

The lambdaPrimeRtool has two external dependencies that need to be installed and added to your systemś path before using the lambdaPrimeR tool

  • MELTING (>= v5.0): used for the calculation of melting temperature of DNA sequences
  • NUPACK (>= v3.1): used to perform the free energy calculations of primer-vector and primer-primer complexes.

NUPACK needs to be compiled from the source code using cmake. To download and install cmake, please follow the instructions here.

Using the lambdaPrimeR package

The lambdaPrimeR primer design process is very simple, and only requires three main functions: read_inputs(), create_primers(), and evaluate_primers(). After the primers are designed, the report_primers() function exports the best primer pairs into a .csv file. In ordert to better understand how the primer design process works, we will breakdown the necessary steps one by one on the next sections.

Loading data

In order to design the primers, we will need only three main inputs: the target gene sequences, the destination vector sequence and the poisition that you want to insert the target gene into the vector. The target and vector sequences can be imported from FASTA and CSV files, you simply need to provide the filepath to the read_inputs().

Let’s import two example files to demostrate how to use the read_inputs() function. The two example files, GFP sequences.fasta and pUC19.fasta are provided with the lambdaPrimeR tool:

#Specify target and vector input files paths:
target_input_example <- system.file('extdata', 'GFP sequence.fasta', package = "lambdaPrimeR")
vector_input_example <- system.file('extdata', 'pUC19.fasta', package = "lambdaPrimeR")

#Reading templates using the read_inputs() function:
inputs <- read_inputs(target = target_input_example, vector = vector_input_example)

The read_inputs() function imports the FASTA files and stores the input information (name, sequence, length, ect) in a Run object. The Run object is an S4 class that represents a \(\lambda\)-PCR reaction, therefore it holds all the information necessary to create the primer pairs. The Run object contains the following slots:

  • Target: a data frame with the target input information
  • Vector: a data frame with the vector input information
  • Primers: an empty data frame that will store the designed primer pairs and relevant thermodynamic data (annealing temp, complex free energies, etc)
  • id: an unique identifier for the current primer design event. This can be provided by you or generated automatically from the input target and vector names (default).

The other main functions of the lambdaPrimeR package interact with the Run objected created by the read_inputs() function.
